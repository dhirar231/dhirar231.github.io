Commandes de Configuration Réseau

Commandes Principales

Cette section présente les commandes de configuration utilisées pour déployer l'infrastructure réseau de TechSolutions SARL.

Configuration OSPF sur RZ1 (Web/Marketing)
RZ1> enable
RZ1# configure terminal
RZ1(config)# interface gigabitethernet0/0
RZ1(config-if)# ip address 172.24.0.1 255.255.192.0
RZ1(config-if)# no shutdown
RZ1(config-if)# exit
RZ1(config)# router ospf 1
RZ1(config-router)# network 172.24.0.0 0.0.63.255 area 0
RZ1(config-router)# passive-interface gigabitethernet0/0
RZ1(config-router)# exit
Configuration OSPF sur RZ2 (IT)
RZ2> enable
RZ2# configure terminal
RZ2(config)# interface gigabitethernet0/0
RZ2(config-if)# ip address 172.24.64.1 255.255.248.0
RZ2(config-if)# no shutdown
RZ2(config-if)# exit
RZ2(config)# router ospf 1
RZ2(config-router)# network 172.24.64.0 0.0.7.255 area 0
RZ2(config-router)# passive-interface gigabitethernet0/0
RZ2(config-router)# exit
Les configurations OSPF pour RZ3 et RZ4 suivent le même modèle avec leurs réseaux respectifs.
Configuration VLAN sur RZ2 (IT - VLAN 20)
! Création du VLAN 20 pour le département IT
RZ2> enable
RZ2# configure terminal
RZ2(config)# vlan 20
RZ2(config-vlan)# name IT_VLAN
RZ2(config-vlan)# exit

! Configuration de l'interface subinterface pour le VLAN 20
RZ2(config)# interface gigabitethernet0/0.20
RZ2(config-subif)# encapsulation dot1Q 20
RZ2(config-subif)# ip address 172.24.64.1 255.255.248.0
RZ2(config-subif)# no shutdown
RZ2(config-subif)# exit
Configuration VLAN sur RZ3 (DB - VLAN 30)
! Création du VLAN 30 pour le département Base de Données
RZ3> enable
RZ3# configure terminal
RZ3(config)# vlan 30
RZ3(config-vlan)# name DB_VLAN
RZ3(config-vlan)# exit

! Configuration de l'interface subinterface pour le VLAN 30
RZ3(config)# interface gigabitethernet0/0.30
RZ3(config-subif)# encapsulation dot1Q 30
RZ3(config-subif)# ip address 172.24.72.1 255.255.254.0
RZ3(config-subif)# no shutdown
RZ3(config-subif)# exit
Configuration DHCP sur RZ1 (Web/Marketing)
! Configuration du pool DHCP pour le VLAN 10
RZ1> enable
RZ1# configure terminal
RZ1(config)# ip dhcp pool VLAN10_WEB
RZ1(dhcp-config)# network 172.24.0.0 255.255.192.0
RZ1(dhcp-config)# default-router 172.24.0.1
RZ1(dhcp-config)# dns-server 8.8.8.8 1.1.1.1
RZ1(dhcp-config)# domain-name techsolutions.local
RZ1(dhcp-config)# lease 7
RZ1(dhcp-config)# exit

! Exclusion des adresses réservées aux serveurs
RZ1(config)# ip dhcp excluded-address 172.24.0.1 172.24.0.9
RZ1(config)# ip dhcp excluded-address 172.24.0.255

Configuration NAT sur RZ1
! Configuration de l'interface externe (Internet)
RZ1> enable
RZ1# configure terminal
RZ1(config)# interface gigabitethernet0/1
RZ1(config-if)# ip address dhcp
RZ1(config-if)# ip nat outside
RZ1(config-if)# no shutdown
RZ1(config-if)# exit

! Configuration de l'interface interne
RZ1(config)# interface gigabitethernet0/0
RZ1(config-if)# ip nat inside
RZ1(config-if)# exit

! Définition des réseaux internes autorisés à utiliser NAT
RZ1(config)# access-list 1 permit 172.24.0.0 0.0.255.255

! Configuration du NAT Overload (PAT)
RZ1(config)# ip nat inside source list 1 interface gigabitethernet0/1 overload
RZ1(config)# exit
Commandes de Vérification
! Vérification de la table de routage
RZ1# show ip route

! Vérification des voisins OSPF
RZ1# show ip ospf neighbor

! Vérification des interfaces
RZ1# show ip interface brief

! Vérification des VLAN
RZ2# show vlan brief

! Vérification des baux DHCP
RZ1# show ip dhcp binding

! Vérification des traductions NAT
RZ1# show ip nat translations

Bonnes Pratiques

Configuration OSPF
  • Utiliser l'area 0 comme backbone
  • Configurer les interfaces passives vers les réseaux locaux
  • Utiliser l'authentification OSPF pour la sécurité
  • Configurer des timers appropriés selon la topologie
Configuration VLAN
  • Documenter les VLANs et leurs utilisations
  • Utiliser des noms explicites pour les VLANs
  • Configurer le VLAN natif sur les trunks
  • Implémenter VTP avec prudence ou utiliser le mode transparent
Configuration DHCP
  • Exclure les adresses des équipements réseau
  • Configurer des durées de bail appropriées
  • Utiliser des serveurs DNS internes si disponibles
  • Configurer des réservations pour les serveurs
Sécurité
  • Configurer des ACLs pour contrôler le trafic inter-VLAN
  • Utiliser des mots de passe forts et les chiffrer
  • Désactiver les services inutiles
  • Configurer le logging et le monitoring
